  OWASP Top 10 2021 Security Review Summary (March 2026)


  A01:2021 – Broken Access Control
   * Status: ✅ IMPROVED
   * Findings: The critical authorization gaps identified in 2024 (maintenance
     resolve and cocurricular photo uploads) have been fixed. Both endpoints now
     verify roles (ra, admin) and resource ownership.
   * Multi-tenancy: The application uses a robust multi-tenant architecture
     where data isolation is enforced at the database level
     (get_tenant_db_for_user), with tenant codes embedded in signed JWTs.
   * Recommendation: Continue auditing new endpoints for consistent use of
     get_tenant_db_for_user.


  A02:2021 – Cryptographic Failures
   * Status: ✅ FIXED
   * Mobile Storage: The mobile app has migrated from AsyncStorage to
     expo-secure-store, which uses platform-native encryption
     (Keychain/Keystore).
   * JWT Security: JWTs are signed with a mandatory JWT_SECRET (validated at
     startup).
   * Recommendation: Access tokens have a 24-hour expiration. While improved,
     implementing a shorter access token (e.g., 1 hour) with a refresh token
     mechanism would further limit the window of opportunity for stolen tokens.


  A03:2021 – Injection
   * Status: ✅ SECURE
   * XSS Prevention: bleach is used consistently for sanitizing user-generated
     content in announcements and messages.
   * File Uploads: validate_image_upload checks magic bytes and scans for
     dangerous patterns (e.g., <script, <?php).
   * SQL/NoSQL: MongoDB queries use parameterized dictionary syntax, preventing
     NoSQL injection.


  A04:2021 – Insecure Design
   * Status: 🟡 PARTIALLY FIXED
   * Account Lockout: Now uses a database-backed store (db.login_attempts),
     making it persistent across server restarts.
   * Rate Limiting: High-priority finding: utils/rate_limit.py still uses an
     in-memory defaultdict. This means rate limits are reset if the server
     restarts.
   * Recommendation: Migrate utils/rate_limit.py to use the RedisBackend already
     implemented in utils/redis_cache.py.


  A05:2021 – Security Misconfiguration
   * Status: ✅ FIXED
   * CORS: Wildcard * has been removed. Origins are now explicitly restricted to
     FRONTEND_URL and authorized mobile app schemes.
   * Headers: SecurityHeadersMiddleware now enforces HSTS, X-Frame-Options:
     DENY, X-Content-Type-Options: nosniff, and a restrictive
     Content-Security-Policy.
   * Attack Surface: OpenAPI/Swagger documentation is disabled in production.


  A06:2021 – Vulnerable and Outdated Components
   * Status: ✅ SECURE
   * Dependencies: Backend (FastAPI 0.128.0) and Frontend (React 19.0.0, Axios
     1.13.5) are up to date.
   * Automation: A scripts/security_audit.sh exists in the codebase to run
     pip-audit and yarn audit during CI/CD.


  A07:2021 – Identification and Authentication Failures
   * Status: ✅ IMPROVED
   * MFA: Multi-Factor Authentication is implemented and required for
     administrative actions.
   * Password Policy: Strong complexity requirements (12+ chars, special chars,
     etc.) and breach checks (utils/password_breach.py) are in place.
   * Session Management: httpOnly cookies are supported for web clients, and
     SecureStore is used for mobile.


  A08:2021 – Software and Data Integrity Failures
   * Status: ✅ SECURE
   * CSV Safety: sanitize_csv_value is implemented and used during bulk user
     imports to prevent CSV formula injection (e.g., escaping =, +, @).


  A09:2021 – Security Logging and Monitoring Failures
   * Status: ✅ FIXED
   * Comprehensive Logging: utils/security_logger.py tracks LOGIN_FAILURE,
     PERMISSION_DENIED, and SUSPICIOUS_ACTIVITY with database persistence.
   * MFA Auditing: MFA setup and verification events are specifically logged.


  A10:2021 – Server-Side Request Forgery (SSRF)
   * Status: ✅ SECURE
   * URL Validation: utils/url_validation.py implements strict checks for
     external photo URLs, blocking internal IPs and metadata service addresses
     (e.g., 169.254.169.254).

  ---

  Final Recommendations


   1. Distributed Rate Limiting: Update utils/rate_limit.py to use Redis.
      Currently, an attacker could bypass rate limits by triggering a server
      restart or if the app is scaled across multiple containers.
   2. Metadata Stripping: Enhance utils/file_validation.py to strip EXIF
      metadata from uploaded images to protect user privacy (location data).
   3. Token Refresh: Implement a refresh token flow to allow for shorter access
      token lifespans (1 hour) without forcing users to re-login daily.
   4. Frontend Token Storage: Investigate moving the React frontend entirely to
      httpOnly cookies for token storage to eliminate the XSS risk associated
      with localStorage.
